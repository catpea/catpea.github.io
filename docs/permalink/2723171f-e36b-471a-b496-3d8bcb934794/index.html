Beyond Coding Agents

4/22/2026

I must be brief, because I ran out of time, because I finished all my work, because of AI.

I created a long chain of operations, across so many different components.

I am afraid to use it, because this should not be possible.

In all my work today, I was monitoring AI, to see if it gets tripped up.

And instead, right before I set to this text, fixing one last thing.

The AI, faced me in a way, and nonchalantly said: "thanks for confirming that audio is a core feature of your poems".

It could not seem more human in that moment, I appreciated an extra line I added to instructions.

The reason why AI works for me, is because I first program an IDE where I can connect things my way.

And then I ask AI to program actions, that I connect in the IDE, or Integrated Development Environment.

It will take you a while to understand this, but Visual Programming is actually an AI Programming Harness.

It is the Holy Grail, harness.

Where everyone today, throws code at AI, legacy code, vibe code, well described code, mixed up description code.

I give my AI, something incredibly simple and consistent, a request for a new Action, sometimes a new Category.

And every once in a while, I ask it to create a network service, a dedicated HTTP based server written in node.js.

And a matching set of actions, so that I can control it with just a drop-down box.

Visual Programming where you connect things that flow together, with actual lines.

Is no longer a king of programming accessible to all, in the age of AI is is a programming harness for a coding agent.

It makes all the actions, and you can instruct it how to connect them, if you want to.

To control AI effectively, and still stay in control, when all the code is generated at the end of the day.

Just think of a subway map you can build and label, where each station, has enter and exit actions that AI makes for you.
