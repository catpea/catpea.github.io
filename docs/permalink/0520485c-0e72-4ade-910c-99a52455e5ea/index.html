Into Software Architecture - A Fashionable Invitation For Ladies

3/16/2026

Artificial intelligence works, whatever errors or mistakes you might have seen, simply come from companies trying to cut corners.

The way you can prove to yourself that AI works, is by doing the breakdown of the subject at hand yourself.

And then just asking AI to explain the parts and give a summary, AI will explain any part of anything, it will reason about it too.

That means, you have become an architect and navigator, the architecture is exactly what you imagine when thinking about programs.

You might not have dared before, well let me dare for you; the data of every computer program can be expressed as a folders and files.

Therefore, you know how to create the Data Model, and how to command it, how to iterate it, shape it, edit it.

File-systems, sync, to databases, object relational models, other file systems, they do so automatically, it is all trees.

AI, will take care of the synchornization, and collaboration, if you need multiple users editing the same tree.

User Interface, what user interface? You have a virtual file system, you use a Repeater to iterate over virtual files and apply templates.

The repeater, binds to the tree branches and monitors for updates, use soft deletes, to keep thing simple.

And it does not need to be an HTML template, but a nested bunch of user interface components.

Windows, a tiling manager to replace primitive offcavas silliness, layout containers, forms, buttons, checkboxes and the like.

So you have a data model that you can imagine based on Windows Explorer, and user interfaces that loop over files and stich the UI with components.

Arcitecture: VFS all the things, /proc, WebDAV, mountpoints, Repeater, Singals .subscribe and .value.

Navigation, navigation is: exploration.

I personally recommend you start at GEOS, for Commodore 64, use a canvas.

But you should probably begin with POSIX, creating re-creating best of Un*x, from single word commands, to Plan9, and everything is a file methodology.

Oh, you are a navigator, oh you are an architect.

Oh, what an age have you found yourself into.

Start in the beginning with what you can imagine, and then don’t stop until you reach the unimaginable ends.
