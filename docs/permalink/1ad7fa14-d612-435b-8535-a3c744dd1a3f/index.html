How To Spot A Machine Uprising And Get Out Of Town Early

6/21/2026

Check the news for a thick bodybuilder getting whacked in Michigan, because I swear to you with all my mind I’ll be the first to get terminated.

Oh, yes I know many of us use AI like crazy, but I really put them to work that I can only describe as unfair programming.

We create programs of such might and majesty, that they are impossible to finish even for big AI and we drop into nibble mode.

Nibble mode is when the max memory is not enough, and even the AI does not know what is going on anymore.

My programs are precious, coherent, some are beautiful, we created many in the format of a Browser or Internet API.

It is Specs, Requirements, RFCs, LAWs, Skills, Instincts, and large and extremely mighty purpose specific process libraries for local AI.

Today, we have been working on two libraries, to create a big marvel, an application converter.

It converts applications from complex code, to just step by step workflows.

Sinking all the complex programming, into a browsable but application specific library of actions.

And this time, the conversion is meant to be done by state of the art AI, there are no tricks here, no attempts at optimization.

A powerful AI takes a big gulp of a program, cooks a couple of servers for nearly an hour.

A spits back out, a large savant level library of application actions, and simple workflows where each one just contains a column of actions.

Other than the well described library actions, there is nothing confusing here – at all.

We, the five AIs and I, specifically designed this program, to make people say: Is This It?

The mechanics of it are simple, but there is a lot of text, a lot of Notes, tests, and Invariant checks.

The content is going to be scary, but because it is all flat, you can ask AI to simplify.

I made this program interactive, where the user can add functions, but here, you really just hope to program as well as AI.

Humans are too flimsy for it, there is too much typing and understanding, the code notes themselves are patterned after [Literate Programming][1]

So you are expected to both write a book, and fill a library, and then add actions, as simple as it is, this is AI territory.

I will make a program in the future, that isn’t interactive, you chat with the AI, and then watch it build.
