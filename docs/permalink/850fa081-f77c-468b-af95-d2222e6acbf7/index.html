A Touch Of Programming With Samurai Porkchops

7/14/2026

I am not going to get technical, but there are three parts to my programming machine.

The first component is an XML programming language, designed by state of the art AI, and for local AI, at the moment Ornith

The second component are advanced User Interface components, they are really advanced, audio, video, visual programming, etc.

And the third component is a coding agent, it is currently being programmed upstairs.

You tell the coding agent what it needs, it will select from a large number of application templates.

And begin its work, the agent, writes in my language only.

I predict being able to create state of the art user interface applications, anything yo have ever heard of, you can make for yourself.

The advanced processing, video, images, music, voice, audio enrichment, that sits in a virtual machine next to this program.

They are called containers, and they contain linux, and really advanced backbends.

AI will know to setup those containers, and keep them updated, all the advanced without al the problems.

I make user interface applications, but to make them manageable, clear.

I needed to create a web application specific language, that packed server and constraint information for AI.

Yesterday I ran a state of the art AI for 13 hours straight, successfully reducing 2,000 lines of dense JavaScript in a test application to 0.

That means AI successfully rewrote the already tiny 2,000 of video editors code, into my language, it was extending the language and eating the code.

I did this to learn how to use AI, and you know, eliminate the need to install other peoples crap on my computer.

But if AI dropped to $600 per computer, which it will never do, you could now choose between owning a 3D printer, or Application Builder.

My software is free, and you should build your own version to practice, but you get some options here.

Aside from fancy things for your Design Portfolio: “always nice, never time”.

This is a path for any young person, to setup an internet server, with monthly billing, all nice and easy in the age of AI.

In general the most precious thing here is having a bunch of stuff, that you feel will sell, and that pushes you forward to learn more.

I have already begun making preparation for next Tuesday, this program, like 4 variation of it before is getting retired soon.

My next one is crafted to AI that can run in a web page, is you have a recent computer.

And let me close this text by telling you how it works, I always brought software to a virtual world with cities or offices.

There AI agents could expand it, and to give a rough and sketchy example, no agent ever needed to learn how to open a window.

But, today I begun work, by treating software as a virtual world already.

Buttons can be taken by the user into their inventory, and dropped elsewhere, through TAKE and DROP actions you know this as Copy and Paste.

But in this virtual world with a verb calculus, you can also USE a button WITH an empty or existing WORKFLOW.

And then inside this workflow, place mores actions such us WRITE contactForm/message INTO emailBox.

This is based on old text adventure games and verb user interface, but you and AI get the same easy language, and you use it with a mouse.

The result looks like a professional application, but the design experience is very much that of playing a game.

The program I am finishing today, id the most you can ruin on a local computer, what I am preparing for next Tuesday, so to speak, is its bare minimum.

Thus, my programming adventure, continues.
